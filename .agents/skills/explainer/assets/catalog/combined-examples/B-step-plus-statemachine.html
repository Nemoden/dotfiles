The side-car pattern. A secondary view that tracks the primary one lets one screen answer both questions at once: what runs (the stepper walking the request path) and what state it produces (the machine lighting the path through RECEIVED → CLASSIFYING → CLASSIFIED → AUTO_REPLIED). The state machine is rarely the star, but as a passenger it is cheap and constantly useful — you never lose sight of the state while watching the code.
