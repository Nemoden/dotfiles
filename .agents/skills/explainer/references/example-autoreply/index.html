Explainer Technique Catalog

A library of interactive techniques for explaining a code change. Each is a self-contained demo built on the SAME content (a fictional support auto-responder "preview" change) so you compare interaction shape, not subject. Think of these like tools in an agentic loop: a future skill picks one (or a few) by name for the job at hand.

12 primitive techniques + 5 combined examples · shared renderer with indentation-safe diffs · dark theme · vanilla JS, no dependencies

Combined examples

Techniques are primitives; they layer. Annotated-source slots into anything that shows code; state-machine and minimap ride along as side-cars; provenance zoom drops in at ground level. These examples compose 2–5 primitives to show when combining beats a single view — and where "integrated" tips into "cluttered".

How a skill would use this: match the reader's question to a technique. "Where does channel come from?" → Value Provenance. "What happens when I run it?" → Runtime Step-Through or Timeline Scrubber. "How is preview different from live?" → Dual-Lens or State-Machine. "I keep getting lost in this big change" → Minimap + Detail or Focus + Context. "Just explain it well in words" → Narrative Walkthrough. Techniques compose — a doc can open with a Spine, let readers zoom via Altitude, and back each claim with Annotated Source.
