Collaborator Swimlanes

Actors are vertical lanes; time flows downward; each numbered row is a message (a horizontal arrow) from one lane to another. This is the call order and who-calls-whom that a static box diagram hides.
Same sequence of calls in both modes. Preview does not add, remove, or reorder a single message — process_ticket never learns it is a preview. All that changes is what each write-message DOES: it becomes an inert no-op that returns a truthy value so the one control-flow path still runs end-to-end. The reads (store.get) and the reasoning (classify_fn) stay real.
